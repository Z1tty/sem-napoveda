Dynamický retargeting (DRTG)

Dynamický retargeting umožňuje automaticky zobrazovat reklamy na konkrétní produkty nebo kategorie, které si uživatel prohlížel. Stačí správně předat parametry v události ViewContent nebo PageView.

Jak dynamický retargeting funguje

SEM automaticky zařadí uživatele do retargetingového seznamu na základě událostí, které odešlete. Nemusíte vytvářet samostatné retargetingové kódy – stačí správně vyplnit parametr content_type a contents v událostech ViewContent nebo PageView.

Sklik pak může automaticky zobrazit reklamu na přesně ten produkt nebo kategorii, kterou uživatel navštívil – bez ručního párování publika s produktovým feedem.

DRTG pro produkty

Pro dynamický retargeting produktů odesílejte události ViewContent nebo PageView. Parametr content_type může být nastaven na 'product' nebo zůstat nevyplněný. Pro přesnější párování s produktovým feedem předejte v poli contents objekt s id produktu – hodnota musí odpovídat ID v produktovém feedu Sklik / Zboží.cz.

Zpracovává se jen první objekt v contents

Pokud v události zasíláte více objektů v poli contents, pro účely DRTG produktů se zpracuje pouze první z nich. Ostatní objekty jsou ignorovány.

JavaScript – ViewContent (detail produktu) 
SEM('track', 'ViewContent', {
  currency: 'CZK',  // podporováno pouze CZK
  value: 1490,
  content_type: 'product',   // volitelné – lze vynechat
  contents: [
    {
      id: 'SKU-12345',        // ID produktu z feedu
      quantity: 1,
      price: 1490
    }
  ]
});

DRTG pro kategorie

Pro dynamický retargeting kategorií použijte content_type: 'product_group' a předejte název kategorie v parametru content_category.

JavaScript – ViewContent (výpis kategorie) 
SEM('track', 'ViewContent', {
  currency: 'CZK',  // podporováno pouze CZK
  value: 0,
  content_type: 'product_group',   // povinné pro DRTG kategorií
  content_category: 'Televizory',  // název kategorie
  contents: []
});

Alternativa pomocí PageView:

JavaScript – PageView 
SEM('track', 'PageView', {
  content_type: 'product_group',
  content_category: 'Televizory'
});

Přehled – kdy použít který content_type

Stránka content_type Povinné parametry Volitelné
Detail produktu product contents[].id contents[].quantity, contents[].price, value
Výpis kategorie / vyhledávání product_group content_category contents[] (volitelně ID produktů ze stránky)
Domovská stránka / ostatní nevyplňovat – –
ID produktu musí odpovídat feedu

Hodnota contents[].id musí přesně odpovídat ID produktu v produktovém feedu provozovny Seznam Nákupy. Jinak nedojde ke správnému párování a dynamická reklama se nezobrazí.

Dynamické retargetingové seznamy

Dynamické retargetingové seznamy vznikají automaticky – pro každou napojenou provozovnu Seznam Nákupy se při jejím vytvoření automaticky vytvoří tři dynamické seznamy:

  • DRTG – Návštěvníci produktů
  • DRTG – Návštěvníci produktů na webech Seznamu
  • DRTG – Návštěvníci kategorií

Podmínky pro sběr dat v těchto seznamech nelze ručně definovat. Stačí správně implementovat události ViewContent nebo PageView s parametry content_type a contents[] – SEM data do dynamických seznamů přiřadí automaticky.

Vlastní RTG seznamy a podmínky

Pokud chcete vytvořit vlastní retargetingové publikum s podmínkami (URL, kategorie, hodnota nákupu, pohlaví…), přejděte do sekce Retargeting.

Implementace přes GTM

Při použití GTM šablony stačí v tagu ViewContent vyplnit pole Content Type (product nebo product_group) a Contents z dataLayer proměnné obsahující pole produktů. Šablona automaticky předá parametry do SEM.

Viz GTM implementace → ViewContent.

Dynamický retargeting přes S2S

Pro S2S měření předejte stejné parametry v sekci event_data payloadu:

JSON – event_data 
"event_data": {
  "currency": "CZK",  // podporováno pouze CZK
  "value": 1490,
  "content_type": "product",
  "contents": [
    {
      "id": "SKU-12345",
      "quantity": 1,
      "price": 1490
    }
  ]
}
