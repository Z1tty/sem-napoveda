Dynamický retargeting (DRTG)

Dynamický retargeting umožňuje automaticky zobrazovat reklamy na konkrétní produkty nebo kategorie, které si uživatel prohlížel. Stačí správně předat parametry v události ViewContent nebo PageView.

Jak dynamický retargeting funguje

SEM automaticky zařadí uživatele do retargetingového seznamu na základě událostí, které odešlete. Nemusíte vytvářet samostatné retargetingové kódy – stačí správně vyplnit parametr content_type a contents v událostech ViewContent nebo PageView.

Sklik pak může automaticky zobrazit reklamu na přesně ten produkt nebo kategorii, kterou uživatel navštívil – bez ručního párování publika s produktovým feedem.

DRTG pro produkty

Pro dynamický retargeting produktů odesílejte události ViewContent nebo PageView. Parametr content_type může být nastaven na 'product' nebo zůstat nevyplněný. Pro přesnější párování s produktovým feedem předejte v poli contents objekt s id produktu – hodnota musí odpovídat ID v produktovém feedu Sklik / Zboží.cz.

Zpracovává se jen první objekt v contents

Pokud v události zasíláte více objektů v poli contents, pro účely DRTG produktů se zpracuje pouze první z nich. Ostatní objekty jsou ignorovány.

JavaScript – ViewContent (detail produktu) 
SEM('track', 'ViewContent', {
  currency: 'CZK',  // podporováno pouze CZK
  value: 1490,
  content_type: 'product',   // volitelné – lze vynechat
  contents: [
    {
      id: 'SKU-12345',        // ID produktu z feedu
      quantity: 1,
      price: 1490
    }
  ]
});

DRTG pro kategorie

Pro dynamický retargeting kategorií použijte content_type: 'product_group' a předejte název kategorie v parametru content_category uvnitř pole contents[].

JavaScript – ViewContent (výpis kategorie) 
SEM('track', 'ViewContent', {
  content_type: 'product_group',
  currency: 'CZK',
  value: 0,
  contents: [{
    content_category: 'Elektronika | Televize | LCD televizory'  // kategorie pro DRTG
  }]
});

Alternativa pomocí PageView:

JavaScript – PageView 
SEM('track', 'PageView', {
  content_type: 'product_group',
  contents: [{
    content_category: 'Elektronika | Televize | LCD televizory'
  }]
});

Přehled – kdy použít který content_type

Stránka content_type Povinné parametry Volitelné
Detail produktu product contents[].id contents[].quantity, contents[].price, value
Výpis kategorie / vyhledávání product_group content_category contents[] (volitelně ID produktů ze stránky)
Domovská stránka / ostatní nevyplňovat – –
ID produktu musí odpovídat feedu

Hodnota contents[].id musí přesně odpovídat ID produktu v produktovém feedu provozovny Seznam Nákupy. Jinak nedojde ke správnému párování a dynamická reklama se nezobrazí.

Dynamické retargetingové seznamy

Dynamické retargetingové seznamy vznikají automaticky – pro každou napojenou provozovnu Seznam Nákupy se při jejím vytvoření automaticky vytvoří tři dynamické seznamy:

  • DRTG – Návštěvníci produktů
  • DRTG – Návštěvníci produktů na webech Seznamu
  • DRTG – Návštěvníci kategorií

Podmínky pro sběr dat v těchto seznamech nelze ručně definovat. Stačí správně implementovat události ViewContent nebo PageView s parametry content_type a contents[] – SEM data do dynamických seznamů přiřadí automaticky.

Vlastní RTG seznamy a podmínky

Pokud chcete vytvořit vlastní retargetingové publikum s podmínkami (URL, kategorie, hodnota nákupu, pohlaví…), přejděte do sekce Retargetingové seznamy.

Implementace přes GTM

Při použití GTM šablony stačí v tagu ViewContent vyplnit pole Content Type (product nebo product_group) a Contents z dataLayer proměnné obsahující pole produktů. Šablona automaticky předá parametry do SEM.

Viz GTM implementace → ViewContent.

Dynamický retargeting přes S2S

Pro S2S měření předejte stejné parametry v sekci event_data payloadu:

JSON – event_data 
"event_data": {
  "currency": "CZK",  // podporováno pouze CZK
  "value": 1490,
  "content_type": "product",
  "contents": [
    {
      "id": "SKU-12345",
      "quantity": 1,
      "price": 1490
    }
  ]
}

Dynamic Retargeting (DRTG)

Dynamic retargeting allows you to automatically show ads for specific products or categories that a user has viewed. All you need to do is pass the correct parameters in a ViewContent or PageView event.

How dynamic retargeting works

SEM automatically adds users to a retargeting list based on the events you send. You do not need to create separate retargeting codes – simply fill in the content_type and contents parameters in the ViewContent or PageView events.

Sklik can then automatically show an ad for the exact product or category the user visited – without manual audience-to-product-feed matching.

DRTG for products

For dynamic retargeting of products, send ViewContent or PageView events. The content_type parameter can be set to 'product' or left empty. For more accurate matching with the product feed, pass an object with the product id in the contents array – the value must match the ID in the Sklik / Seznam Shopping product feed.

Only the first object in contents is processed

If you send multiple objects in the contents array, only the first one is processed for DRTG product purposes. The remaining objects are ignored.

JavaScript – ViewContent (product detail) 
SEM('track', 'ViewContent', {
  currency: 'CZK',  // only CZK is supported
  value: 1490,
  content_type: 'product',   // optional – can be omitted
  contents: [
    {
      id: 'SKU-12345',        // product ID from the feed
      quantity: 1,
      price: 1490
    }
  ]
});

DRTG for categories

For dynamic retargeting of categories, use content_type: 'product_group' and pass the category name in the content_category parameter inside the contents[] array.

JavaScript – ViewContent (category listing) 
SEM('track', 'ViewContent', {
  content_type: 'product_group',
  currency: 'CZK',
  value: 0,
  contents: [{
    content_category: 'Elektronika | Televize | LCD televizory'  // category for DRTG
  }]
});

Alternative using PageView:

JavaScript – PageView 
SEM('track', 'PageView', {
  content_type: 'product_group',
  contents: [{
    content_category: 'Elektronika | Televize | LCD televizory'
  }]
});

Overview – when to use which content_type

Page content_type Required parameters Optional
Product detail product contents[].id contents[].quantity, contents[].price, value
Category listing / search product_group content_category contents[] (optionally product IDs from the page)
Homepage / other leave empty – –
Product ID must match the feed

The value of contents[].id must exactly match the product ID in the product feed of the store linked to Seznam Shopping. Otherwise the matching will fail and the dynamic ad will not be shown.

Dynamic retargeting lists

Dynamic retargeting lists are created automatically – for each store linked to Seznam Shopping, three dynamic lists are automatically created when the store is connected:

  • DRTG – Product visitors
  • DRTG – Product visitors on Seznam websites
  • DRTG – Category visitors

The conditions for data collection in these lists cannot be defined manually. Simply implement the ViewContent or PageView events correctly with the content_type and contents[] parameters – SEM will assign the data to the dynamic lists automatically.

Custom retargeting lists and conditions

If you want to create a custom retargeting audience with conditions (URL, category, purchase value, gender…), go to the Retargeting lists section.

Implementation via GTM

When using the GTM template, simply fill in the Content Type field (product or product_group) and Contents from a dataLayer variable containing the product array in the ViewContent tag. The template will automatically pass the parameters to SEM.

See GTM implementation → ViewContent.

Dynamic retargeting via S2S

For S2S measurement, pass the same parameters in the event_data section of the payload:

JSON – event_data 
"event_data": {
  "currency": "CZK",  // only CZK is supported
  "value": 1490,
  "content_type": "product",
  "contents": [
    {
      "id": "SKU-12345",
      "quantity": 1,
      "price": 1490
    }
  ]
}
