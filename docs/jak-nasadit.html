Nasazení a přechod na SEM

Průvodce platí jak pro nové inzerenty, tak pro ty, kteří přecházejí ze stávajících Sklik skriptů. Po celou dobu nasazení běží stará i nová měření souběžně – bez výpadku dat.

Zvolte metodu implementace

Všechny čtyři metody jsou rovnocenné z pohledu nasbíraných dat – liší se pouze způsobem nasazení.

Metoda Kdo ji použije Přístup ke kódu webu Technická náročnost
Přímý skript Vývojář s přístupem do HTML šablon Vyžadován Střední – nutné přidat kód na každou stránku
Google Tag Manager Marketér nebo vývojář s GTM Není potřeba (jen GTM) Nízká – šablona v GTM galerii, bez kódování
Doplňky pro platformy Provozovatel e-shopu na platformě s SEM doplňkem Není potřeba (doplněk platformy) Velmi nízká – stačí zadat SEM ID
Server-to-Server (S2S) Vývojář s vlastním backendem Vyžadován (BE + FE) Vysoká – HTTP volání z vašeho serveru

Postup nasazení

  1. Nasazení nových kódů

    Nasaďte SEM napřímo, pomocí připravené GTM šablony nebo ho aktivujte ve vaší webové platformě.

    • Přímý skript – vložení kódu přímo do HTML stránky
    • Google Tag Manager – přes GTM šablonu nebo vlastní HTML tagy
    • Doplněk e-commerce platformy
    • Server-to-Server (S2S) – pokročilé měření z backendu
    E-shopy napojené na Zboží.cz

    Pro všechny události na webu používejte kombinované SEM ID provozovny. Zajistíte tak správné odesílání konverzí za účelem získání zpětné vazby (recenzí produktů) ke konkrétní provozovně.

  2. Migrace konverzí na SEM v účtu Pouze při přechodu

    Správa měření automaticky připraví vaše stávající aktivní konverze k migraci. Většinu konverzí lze převést 1:1. V případě duplicit nebo nepodporovaného typu vás Sklik vyzve k ručnímu nastavení. Konverze, které nechcete migrovat, lze ponechat v přechodném stavu nebo odstranit – do nového měření se pak nepřevedou. Více ve Správě konverzí.

  3. Vytvoření nových konverzních SEM událostí Volitelný

    Další akce uživatelů na vašem webu, které budete chtít v Skliku sledovat, si můžete nastavit předem v Správě měření.

  4. Test SEMu v sandboxu Doporučený

    Nezávazně si ověřte, že všechny události posíláte správně, abyste nepřišli o žádná data. Viz Testování (Sandbox).

  5. Zapnutí SEMu

    Ve Správě měření aktivujte Začít používat SEM. Od této chvíle pro měření konverzí a zařazení publik do retargetingu (RTG) využíváte výhradně události SEM. Rozhraní Skliku vás procesem provede a upozorní na případné nedokončené kroky.

    Pokud máte kampaně s aktivní optimalizací na konverze (ONK), systém vás po překliku na Začít používat SEM vyzve, abyste u každé z nich nastavili, na které SEM konverze se má kampaň optimalizovat.

    Přepnutí nelze vrátit zpět

    Po potvrzení přechodu se staré měřicí kódy přestanou zpracovávat. Ujistěte se, že máte nastavené SEM konverze a ověřená data v Sandboxu.

  6. Vytvoření nových RTG událostí Volitelný

    Z pokročilých údajů, které posíláte pomocí SEM, si můžete vytvořit více retargetingových publik.

  7. Závěrečný úklid webu Pouze při přechodu

    Z webu odstraňte staré retargetingové a konverzní skripty, které už nebudete potřebovat. Jejich ponechání nezpůsobí chybu, ale zbytečně zatěžuje stránku.

Deployment & migration to SEM

This guide applies both to new advertisers and to those migrating from existing Sklik scripts. Throughout the deployment, both the old and new measurement run in parallel – with no data loss.

Choose an implementation method

All four methods collect the same data – they differ only in how they are deployed.

Method Who uses it Website code access Technical complexity
Basic script Developer with access to HTML templates Required Medium – code must be added to each page
Google Tag Manager Marketer or developer with GTM Not required (GTM only) Low – template in GTM gallery, no coding
Platform plugins E-shop operator on a platform with a SEM plugin Not required (platform plugin) Very low – just enter your SEM ID
Server-to-Server (S2S) Developer with a custom backend Required (BE + FE) High – HTTP calls from your server

Deployment steps

  1. Deploy the new code

    Deploy SEM directly, using the prepared GTM template, or activate it in your e-commerce platform.

    • Basic script – embed the code directly into the HTML page
    • Google Tag Manager – via GTM template or custom HTML tags
    • E-commerce platform plugin
    • Server-to-Server (S2S) – advanced backend measurement
    E-shops connected to Seznam Shopping

    For all events on your website, use the combined store outlet SEM ID. This ensures conversions are correctly attributed to the specific store outlet for product review collection.

  2. Migrate conversions to SEM in your account Migration only

    Event Management automatically prepares your existing active conversions for migration. Most conversions can be transferred 1:1. In case of duplicates or unsupported types, Sklik will prompt you to configure them manually. Conversions you do not want to migrate can be left in a transitional state or deleted – they will not be carried over to the new measurement. See Conversion management for more details.

  3. Create new SEM conversion events Optional

    Additional user actions on your website that you want to track in Sklik can be set up in advance in Event Management.

  4. Test SEM in sandbox Recommended

    Verify that all events are being sent correctly before going live, so you don't lose any data. See Testing (Sandbox).

  5. Enable SEM

    In Event Management, activate Start using SEM. From this point on, conversions and retargeting audience membership are measured exclusively through SEM events. The Sklik interface will guide you through the process and alert you to any incomplete steps.

    If you have campaigns with active conversion optimization (CBO), after clicking Start using SEM the system will prompt you to select which SEM conversions each campaign should optimize for.

    This switch cannot be reversed

    Once you confirm the transition, the old measurement codes will stop being processed. Make sure you have your SEM conversions configured and your data verified in Sandbox.

  6. Create new retargeting events Optional

    Using the rich data you send via SEM, you can create additional retargeting audiences.

  7. Clean up your website Migration only

    Remove the old retargeting and conversion scripts from your website – they are no longer needed. Leaving them in place won't cause errors, but they add unnecessary load to your pages.
