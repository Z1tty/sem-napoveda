Objekt produktů – contents

Pole contents popisuje jednotlivé produkty v transakci. Předává se v událostech ViewContent, AddToCart, AddToWishlist, InitiateCheckout, AddPaymentInfo a Purchase.

Parametry objektu

Každý objekt v poli popisuje jeden produkt. Pole může obsahovat libovolný počet objektů.

ParametrTypDoporučenostPříklad hodnotyPopis
idstring Povinný u Purchase 'ABC12345' Unikátní identifikátor položky v e-shopu.
quantitynumber Povinný u Purchase 2 Počet kusů daného produktu v košíku / objednávce.
unit_pricenumber Povinný u Purchase 33490 Cena za jeden kus v CZK včetně DPH.
content_namestring Povinný u Purchase 'iPhone 15 Pro Max' Libovolný název. Lze použít jako podmínku retargetingu.
content_categorystring Doporučený 'Elektronika | Mobilní telefony | Apple' Kompletní cesta kategorie z vašeho XML feedu. Jako oddělovač úrovní slouží svislá čára | (pipe). Předávejte stejnou hodnotu, jakou máte uvedenou u nabídek v XML feedu – celý řetězec pak lze použít jako přesnou podmínku pro retargeting nebo konverze.

Příklad

Příklad – contents[] 
contents: [
  {
    id: 'ABC12345',
    quantity: 1,
    unit_price: 33490,            // s DPH
    content_name: 'iPhone 15 Pro Max',
    content_category: 'Elektronika | Mobilní telefony | Apple'
  },
  {
    id: 'KRYT999',
    quantity: 2,
    unit_price: 500,              // s DPH
    content_name: 'Ochranný kryt'
  }
]
Hodnota value vs. součet contents

Parametr value na úrovni události je celková hodnota produktů bez DPH. Parametry unit_price v contents[], delivery_price a other_costs jsou včetně DPH. value nemusí nutně odpovídat součtu unit_price × quantity přepočítanému na základ daně – může zahrnovat slevy nebo jiné úpravy. Detailní rozpad košíku je v contents[].

Product object – contents

The contents array describes the individual products in a transaction. It is passed in the ViewContent, AddToCart, AddToWishlist, InitiateCheckout, AddPaymentInfo and Purchase events.

Object parameters

Each object in the array describes one product. The array may contain any number of objects.

ParameterTypeRequirementExample valueDescription
idstring Required for Purchase 'ABC12345' Unique identifier of the item in the e-shop.
quantitynumber Required for Purchase 2 Number of units of the product in the cart / order.
unit_pricenumber Required for Purchase 33490 Price per unit in CZK incl. VAT.
content_namestring Required for Purchase 'iPhone 15 Pro Max' Any name. Can be used as a retargeting condition.
content_categorystring Recommended 'Elektronika | Mobilní telefony | Apple' Full category path from your XML feed. Use a vertical bar | (pipe) as the level separator. Pass the same value as listed for offers in the XML feed – the full string can then be used as an exact condition for retargeting or conversions.

Example

Example – contents[] 
contents: [
  {
    id: 'ABC12345',
    quantity: 1,
    unit_price: 33490,            // incl. VAT
    content_name: 'iPhone 15 Pro Max',
    content_category: 'Elektronika | Mobilní telefony | Apple'
  },
  {
    id: 'KRYT999',
    quantity: 2,
    unit_price: 500,              // incl. VAT
    content_name: 'Ochranný kryt'
  }
]
value vs. sum of contents

The value parameter at event level is the total value of products excl. VAT. The unit_price parameters in contents[], delivery_price and other_costs are incl. VAT. value does not necessarily equal the sum of unit_price × quantity converted to the tax base – it may include discounts or other adjustments. The detailed cart breakdown is in contents[].
